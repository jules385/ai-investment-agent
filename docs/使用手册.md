# A股AI投研系统 — 使用手册

> **版本**：v0.1.0 | **更新日期**：2026-06-19
> **适用人群**：买方投资分析师 | **运行环境**：Claude Code + DeepSeek V4 Pro

---

## 一、系统概述

本系统是一套运行在 Claude Code 上的**多智能体投研协作平台**。它模拟买方投研团队的完整工作流——从行业判断到买卖决策，由 8 个 AI 角色分工协作完成。

### 核心理念

> **研究驱动决策，纪律约束情绪。四维交叉验证，数据支撑每一个观点。**

---

## 二、团队角色

系统共有 **8 个 AI 角色**：

### 2.1 分析层（4 位专业分析师）

| 角色 | 核心问题 | 数据来源 | 关键产出 |
|------|---------|---------|---------|
| **基本面分析师** | 这生意值不值得投？ | finance-data MCP + WebSearch | 行业产业链拆解 → 三端增长逻辑 → 财务交叉验证 → 估值判断 → 白名单决策矩阵 |
| **筹码面分析师** | 大资金在买还是卖？ | finance-data MCP（6工具） | 股东结构变动 + 主力资金流向 + 北向+融资交叉 + 龙虎榜 + 筹码分布五维合力 |
| **技术面分析师** | 什么价买、什么价卖？ | tech-analysis MCP（10工具） | 多周期趋势定位 + 13项指标扫描 + 买卖信号 + 价格锚点(买入/止损/止盈) |
| **情绪面分析师** | 现在该贪婪还是恐惧？ | MCP + WebSearch | 市场情绪 + 个股舆情 + 散户行为逆向解读 → 情绪温度计(0-100) |

### 2.2 协调与质量层（2 位）

| 角色 | 职责 | 何时工作 |
|------|------|---------|
| **总分析师（Chief）** | 编排四位分析师并行工作 → 验证输出完整性 → 强制QA → 矛盾检测 → 市况自适应加权 → 合成投资建议 | 所有分析任务的核心调度者 |
| **数据分析师（QA）** | 逐条扫描报告观点 → MCP实时回测验证数据 → 自主补充缺失数据 → 生成缺口清单 → 路由回原分析师修订 | 每份子报告完成后、总分析师合成前（强制自动调用） |

### 2.3 辅助层（2 位）

| 角色 | 职责 | 调用方式 |
|------|------|---------|
| **AI美化师** | 将 Markdown 报告转化为高可读性 HTML，自动将 ASCII 产业链拆解/流程图转换为彩色可视化组件 | 用户说"美化一下"时工作 |
| **组合管理员** | 维护三个组合文件（持仓/白名单/黑名单）中的研究记录，每次分析完成后自动追加 | 自动调用，也可手动触发"查看组合" |

## 三、工作流总览

```
用户发起分析
      │
      ▼
总分析师判断周期
      │
      ├── /analyze-initial     → 初次覆盖（四维深度）
      ├── /analyze-weekly      → 周度跟踪（边际监控+快速扫描）
      ├── /analyze-monthly     → 月度跟踪（4周趋势提炼+月频数据）
      ├── /analyze-quarterly   → 季度跟踪（三表联动财务验证）
      └── /analyze-annual      → 年度跟踪（战略复盘+全面刷新）
      │
      ▼
四个分析师并行启动（Subagent）→ 各自调用MCP+WebSearch → 自行写入磁盘
      │
      ▼
数据分析师 QA → MCP实时回测 → 自主补充 → 通行证YAML
      │
      ▼
总分析师合成 → 信号汇总 → 矛盾分析 → 投资决策
      │
      ▼
Markdown 报告 → （/beautify-report）→ HTML 报告
```

**QA 强制执行**：QA 不是可选项。Chief 必须等待所有 QA 返回通行证后才能进入合成。CONDITIONAL_PASS 需要路由回原分析师修订后重新 QA。

## 四、分析周期详解

### 4.1 初次覆盖（/analyze-initial）

**目标**：判断股票是否进入白名单

基本面执行完整五步法：行业产业链拆解（≥12子环节）→ 供给端/需求端/竞争格局三端分析 → 5年财务验证 → DCF三情景估值 → 六维白名单决策矩阵。筹码面/技术面/情绪面同步深度模式。

**输出**：白名单评级（🟢核心 / 🟡卫星 / ⚪观察 / 🔴排除）+ 合理买入区间。

### 4.2 周度跟踪（/analyze-weekly）

**目标**：快速判断本周是否需要行动

基本面做5轮专项搜索（政策/供需/公告/产业链/券商），每条新闻按三维度量化打分（直接性×幅度×概率）。筹码面/技术面/情绪面做快速扫描。

**特色**：支持组合批量模式（`/analyze-portfolio-weekly`），同行业股票共享行业新闻搜索，Token消耗降低60%+。

### 4.3 月度跟踪（/analyze-monthly）

**目标**：基于4周累积数据判断趋势方向（周度 Plus）

基本面汇总过去4周的🟠🔴事件，提炼趋势信号，补充月频专属数据（PMI/行业景气/可比月报）。筹码面汇总4次周度资金流+北向月度净流。技术面做周线+月线趋势+信号命中率统计。情绪面绘制4周温度曲线。

### 4.4 季度跟踪（/analyze-quarterly）

**目标**：用最新财报三张表联动验证投资逻辑

基本面执行7步三表联动分析：利润表（增长质量+费用拆解+非经常性损益）→ 资产负债表（增长代价+周转效率+商誉风险）→ 现金流量表（利润含金量+FCF）→ 三表联动勾稽验证 → 估值更新 → 逻辑验证 → 评级建议。筹码面/技术面/情绪面简略模式。

**升级/降级决策**：系统内置量化规则——毛利率连续2季下滑>3pct、营收同比<15%、ROIC-WACC收窄>3pct等触发降级；营收+净利双超10%+PEG<0.5等触发升级。

### 4.5 年度跟踪（/analyze-annual）

**目标**：战略复盘，决定是否继续持有

基本面用年报完整数据做全年财务回顾（预测 vs 实际），重评行业竞争格局，更新白名单决策矩阵。筹码面回顾全年资金演变。技术面做全年信号命中率审计。情绪面定位年度情绪周期。

**与初次覆盖的区别**：不重做产业链拆解，但用一年的实际数据重新验证每一层逻辑。输出持仓决策（继续持有/加仓/减仓/清仓）而非白名单评级。

## 五、如何使用

### 5.1 斜杠命令速查

| 命令 | 用途 | 示例 |
|------|------|------|
| `/analyze-initial 代码 名称` | 初次覆盖 | `/analyze-initial 002414 高德红外` |
| `/analyze-weekly` | 周度跟踪（遍历白名单） | `/analyze-weekly` |
| `/analyze-weekly 代码` | 周度跟踪（指定标的） | `/analyze-weekly 002414` |
| `/analyze-monthly 代码` | 月度跟踪 | `/analyze-monthly 002414` |
| `/analyze-quarterly 代码` | 季度跟踪 | `/analyze-quarterly 002414` |
| `/analyze-annual 代码` | 年度战略复盘 | `/analyze-annual 002414` |
| `/analyze-portfolio-weekly` | 组合批量周度跟踪 | `/analyze-portfolio-weekly` |
| `/beautify-report 代码-名称` | 生成HTML报告 | `/beautify-report 002414-高德红外` |

也可用自然语言触发，如"对金风科技做初次覆盖""美���一下新易盛的报告"。

### 5.2 组合管理

| 操作 | 方式 |
|------|------|
| 查看组合 | "查看组合" / "组合概览" |
| 查看研究历史 | "最近分析过哪些标的" |
| 跟踪超期提醒 | "哪些标的该跟踪了" |
| 记录研究结果 | 每次分析完成后自动追加 |

### 5.3 初次覆盖后建立跟踪节奏

```
每周末：/analyze-portfolio-weekly    ← 组合批量周度跟踪（推荐）
或每标的逐一：/analyze-weekly
每月初：/analyze-monthly [标的]      ← 月度趋势确认
财报季后：/analyze-quarterly [标的]  ← 季度三表联动
年末：/analyze-annual [标的]         ← 年度战略复盘
```

## 九、核心优势

**1. 多维度交叉验证** — 四位分析师从完全不同的数据源和分析框架独立得出结论。基本面看"值不值"，情绪面看"怕不怕"，筹码面看"谁在买"，技术面看"何时买"。

**2. 数据驱动决策** — 40+ 量化指标覆盖供给端/需求端/竞争格局。每个观点必须有数据支撑，数据分析师对每份报告逐条核实，QA强制不可跳过。

**3. 完整周期体系** — 从初次覆盖到年度复盘，5个分析周期覆盖投资的完整生命周期。每个周期有专属分析模式和输出模板。

**4. 零成本数据基础设施** — 全部基于 AKShare 免费开源数据，无需 Bloomberg/Wind。自建 MCP Server 内置请求节流和自动回退。

**5. 模块化可扩展** — 8 个 Skill + 2 个 MCP Server 均为独立模块。可随时新增数据源、分析维度、替换底层 LLM。

**6. 知识沉淀与复用** — 每个标的独立存档，标准化 CSV + Markdown 格式。决策日志记录每一次操作的理由，事后可复盘。

## 十、快速上手

**第一步：安装**
```bash
git clone https://github.com/jules385/ai-investment-agent.git
cd ai-investment-agent
pip install -r requirements.txt
python install.py
```

**第二步：配置**
1. 编辑 `~/.claude/settings.json`，填入 API Key（参考 `workspace-template/.claude/settings.json.template`）
2. 复制 `workspace-template/` 到工作目录（如 `~/ai-investment/`）
3. 在白名单组合中添加标的

**第三步：初次覆盖**
```
/analyze-initial 002414 高德红外
```

**第四步：持续跟踪 + 生成报告**
```
/analyze-portfolio-weekly              ← 每周批量跟踪
/analyze-monthly 002414                ← 月度趋势确认
/analyze-quarterly 002414              ← 季度三表验证
/beautify-report 002414-高德红外        ← 生成 HTML
```
