# A股二级投研 AI Agent 系统 —— 使用手册

> **版本**：v0.0.0 | **更新日期**：2026-06-17
> **适用人群**：买方投资分析师 | **运行环境**：Claude Code + DeepSeek V4 Pro

---

## 一、系统概述

本系统是一套运行在 Claude Code 上的**多智能体投研协作平台**。它模拟买方投研团队的完整工作流——从行业判断到买卖决策，由 7 个 AI 角色分工协作完成。

### 核心理念

> **研究驱动决策，纪律约束情绪。四维交叉验证，数据支撑每一个观点。**

### 适合谁用

- 个人投资者需要系统化的投研流程
- 买方分析师需要快速覆盖多标的
- 想用 AI 辅助而非替代投资决策的人

---

## 二、团队角色

系统共有 **7 个 AI 角色**，分为分析层、质量层和展示层：

### 2.1 分析层（4 位专业分析师）

| 角色 | 核心问题 | 数据来源 | 关键产出 |
|------|---------|---------|---------|
| **基本面分析师** | 这生意值不值得投？ | AKShare + WebSearch | 行业判断 → 增长逻辑(供给/需求/竞争) → 财务验证 → 白名单评级 |
| **情绪面分析师** | 现在该贪婪还是恐惧？ | AKShare + WebSearch | 情绪温度计(0-100) + 左侧/右侧策略建议 |
| **筹码面分析师** | 大资金在买还是卖？ | AKShare (12个函数) | 资金合力方向 + 对手盘压力 + 建仓/出货阶段 |
| **技术面分析师** | 什么价买、什么价卖？ | tech-analysis MCP (10工具) | 买卖信号 + 价格锚点(买入/止损/止盈) |

### 2.2 协调层（2 位）

| 角色 | 职责 | 何时工作 |
|------|------|---------|
| **总分析师** | 编排四位分析师并行工作，矛盾检测，市况自适应加权，输出最终投资建议 | 所有分析报告完成后 |
| **数据分析师** | 逐条检查报告数据充分性，自主补充缺失数据，生成缺口清单，用户补数据后路由回原分析师修订 | 每份报告完成后、总分析师合成前 |

### 2.3 展示层（1 位，按需调用）

| 角色 | 职责 | 调用方式 |
|------|------|---------|
| **AI美化师** | 将 Markdown 报告转化为高可读性 HTML | 用户说"美化一下"时才工作 |

## 三、工作流全景

```
用户发起分析请求
       │
       ▼
总分析师判断周期类型
       │
       ├── 初次覆盖 → 基本面(深度) + 筹码面(深度) + 技术面(简要)
       ├── 周度跟踪 → 基本面(边际) + 情绪面 + 筹码面 + 技术面
       ├── 月度跟踪 → 四维全开
       ├── 季度跟踪 → 四维全开(重点财务验证)
       ├── 半年跟踪 → 四维全开(重点逻辑重新验证)
       └── 年度跟踪 → 四维全开(全面刷新)
       │
       ▼
四位分析师并行工作（Subagent）
       │
       ▼
数据分析师 QA —— 逐条检查数据充分性 → 自主补充 → 缺口清单
       │
       ▼
总分析师合成 —— 矛盾检测 → 市况加权 → 投资建议(价格+仓位+止损)
       │
       ▼
输出：Markdown 报告 → （用户说"美化"后）→ HTML 报告
```

## 四、两大工作轨道

### Track A：初次覆盖

**目标**：判断股票是否进入投资组合白名单

```
Step 1: 行业判断（产业链图谱 + TAM/SAM/SOM + 驱动力与瓶颈）
   └── 不通过 → 终止覆盖
Step 2: 增长逻辑分析（供给端/需求端/竞争格局 → 三端信号打分 -3~+3）
Step 3: 财务验证（3-5年趋势 + 财务-逻辑一致性检验）
Step 4: 估值判断（PE分位数 + 可比 + DCF区间）
Step 5: 白名单决策矩阵（六维加权 → 🟢核心 / 🟡卫星 / ⚪观察 / 🔴排除）
```

**分析方法**：Step 2 严格遵循《增长逻辑分析方法手册》，从供给端（产能/成本/研发）、需求端（渗透率/粘性/定价）、竞争格局（市占率/ROIC-WACC/进入壁垒）三个方向进行数据驱动的量化打分。

### Track B：持续跟踪

**五级周期体系**：

| 周期 | 频率 | 核心问题 | 参与分析师 |
|------|------|---------|-----------|
| **周度** | 每周六 | "这周发生了什么需要行动的事？" | 全四维(基本面为边际监控) |
| **月度** | 每月初 | "中期趋势强化还是弱化？" | 全四维 |
| **季度** ⭐ | 财报季后 | "财报验证还是证伪了投资逻辑？" | 全四维(重点财务) |
| **半年** | 年中年末 | "行业格局有没有结构性变化？" | 全四维(重点逻辑重验) |
| **年度** | 年末 | "今年的α来自哪里？明年机会在哪？" | 全四维(全面刷新) |

## 六、投资组合结构

| 层级 | 名称 | 数量 | 仓位上限 | 定义 |
|------|------|------|---------|------|
| 🟢 核心层 | 核心持仓 | 3-5 只 | 单只 ≤20% | 高确信度、长持有期（评分 ≥4.0） |
| 🟡 卫星层 | 卫星持仓 | 5-10 只 | 单只 ≤10% | 中短期机会、主题投资（评分 3.0-3.9） |
| ⚪ 观察层 | 观察名单 | 不限 | 0% | 研究覆盖中、等待买点（评分 2.0-2.9） |
| 🔴 黑名单 | 排除 | — | 0% | 存在致命缺陷（评分 <2.0） |

## 九、核心优势

### 1. 多维度交叉验证
四位分析师从完全不同的数据源和分析框架独立得出结论。基本面看"值不值"，情绪面看"怕不怕"，筹码面看"谁在买"，技术面看"何时买"。四维交叉，避免单一维度偏见。

### 2. 数据驱动决策
《增长逻辑分析方法手册》定义了供给端/需求端/竞争格局三个维度共 40+ 量化指标及其分析阈值。每个观点必须有数据支撑，每个趋势必须有3-5年历史数据验证。数据分析师对每份报告进行数据质量检查，缺口透明呈现。

### 3. 标准化流程，可回溯可复盘
每次分析遵循固定工作流，每个标的独立存档。初次覆盖、周度/月度/季度/半年/年度跟踪各有标准模板。决策日志记录每一次操作的理由，事后可复盘。

### 4. 零成本数据基础设施
全部基于 AKShare 免费开源数据，无需 Bloomberg/Wind 等昂贵终端。1000+ API 覆盖行情、财务、资金流向、龙虎榜、筹码分布、融资融券等。

### 5. 模块化可扩展
7 个分析师 Skill + 63 个金融 Skill + 2 个 MCP Server 均为独立模块。可随时新增数据源（Wind/同花顺）、新增分析维度（ESG/量化因子）、替换底层 LLM。

### 6. 知识沉淀与复用
每个标的的研究数据以标准化格式存档（CSV序列+Markdown报告），形成可检索的研究知识库。新覆盖标的时可直接复用行业模板和可比公司数据，研究效率持续提升。

## 十、快速上手

### 第一步：安装

```bash
git clone https://github.com/jules385/ai-investment-agent.git
cd ai-investment-agent
pip install -r requirements.txt
python install.py
```

### 第二步：配置

1. 复制并编辑 `~/.claude/settings.json`（参考 `workspace-template/.claude/settings.json.template`）
2. 填入你的 API Key
3. 将 `workspace-template/` 复制到你的工作目录（如 `~/ai-investment/`）

### 第三步：初次覆盖一个标的
```
/analyze-initial [股票代码] [股票名称]
```

### 第四步：建立跟踪节奏
```
每周六：/analyze-weekly
每月初：/analyze-monthly [标的]
财报后：/analyze-quarterly [标的]
```

### 第五步：需要时美化
```
/beautify-report [代码]-[名称]
```
