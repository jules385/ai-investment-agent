# A股AI投研系统 使用手册

> 版本：v0.4.1  
> 运行环境：Codex + MCP + Obsidian  
> 适用人群：买方投资分析师、个人研究者

## 1. 系统概览

本系统是一套运行在 Codex 中的多智能体投研协作平台。它模拟买方投研团队的工作流：四个领域分析师并行研究，数据 QA 交叉验证，首席分析师合成结论，最后沉淀为 Markdown 研报和 Obsidian 知识库。

核心原则：研究驱动决策，纪律约束情绪，四维交叉验证，数据支撑观点。

## 2. 安装与配置

```bash
pip install -r requirements.txt
copy .mcp.json.template .mcp.json
python tools/self-test.py
```

把 `.mcp.json` 中的 `{{PYTHON_PATH}}` 和 `{{REPO_PATH}}` 替换为本机 Python 路径和项目路径。

当前 Codex + Obsidian 主流程不依赖外部 LLM API。`.env.example` 只给历史解析模块或个人扩展使用，真实 `.env` 不要提交。

## 3. 常用命令

| 命令 | 用途 | 示例 |
|---|---|---|
| `/analyze-initial 代码 名称` | 初次覆盖 | `/analyze-initial 002463 沪电股份` |
| `/analyze-weekly 代码` | 周度跟踪 | `/analyze-weekly 002463` |
| `/analyze-monthly 代码` | 月度跟踪 | `/analyze-monthly 002463` |
| `/analyze-quarterly 代码` | 季度跟踪 | `/analyze-quarterly 002463` |
| `/analyze-annual 代码` | 年度复盘 | `/analyze-annual 002463` |
| `/analyze-daily` | A股投研日报 | `/analyze-daily` |
| `/analyze-portfolio-weekly` | 组合批量跟踪 | `/analyze-portfolio-weekly` |
| `/archive-research` | 外部研报入库 | `/archive-research reports/x.md` |
| `/beautify-report` | 生成 HTML 报告 | `/beautify-report 002463-沪电股份` |

## 4. 初次覆盖与跟踪

```text
用户发起分析
  -> analyst-chief 判断周期
  -> 四个分析师并行调用 MCP 与公开信息
  -> analyst-data-qa 交叉验证
  -> analyst-chief 合成研报
  -> tools/archive-to-vault.py 入库 Obsidian
```

| 周期 | 模式 | 产出重点 |
|---|---|---|
| 初次覆盖 | 深度 | 建立基本面、技术面、筹码面、情绪面和投资决策基线 |
| 周度 | 快速扫描 | 边际变化、交易信号、跟踪任务 |
| 月度 | 趋势确认 | 趋势是否延续、逻辑是否强化 |
| 季度 | 财务核心 | 财务验证、业绩差异、估值重估 |
| 年度 | 战略复盘 | 全年复盘、行业重评、长期假设 |

## 5. 研究成果入库

研报默认写入：

```text
reports/stocks/<代码-名称>/<周期>/
```

入库分两步：

```bash
python tools/archive-to-vault.py <报告路径> <代码>-<名称> <行业名> --emit-plan reports/archive-plan-<代码>.json
```

Codex 根据报告内容填写分类计划后执行：

```bash
python tools/archive-to-vault.py <报告路径> <代码>-<名称> <行业名> --plan reports/archive-plan-<代码>.json
```

入库规则：

- 初次覆盖完整建立四面基线和投资决策。
- 后续跟踪默认只在 `投资决策.md` 记录结论和原文指引。
- 只有重大逻辑变化才更新四面文件或行业知识库。
- `原始报告/` 只做 Markdown 原文复制，由 Python 执行，不需要 AI 判断。

## 6. 日报流程

日报分析区间为指令发出当天以及上一天。日报是新闻整理和影响判断，不默认调用行情、资金、估值或技术指标 MCP。

```text
Codex 调用新闻/公告/政策 MCP 或可信 web 来源采集证据
  -> reports/daily/YYYY-MM-DD/evidence.json
  -> tools/daily_score.py
  -> Codex 做重要性判断与影响链推理
  -> tools/daily_report.py
  -> tools/daily_to_vault.py
  -> obsidian-vault/日报/YYYY-MM-DD.md
```

如果只运行 Python 脚本，`daily_collect.py` 只能接收和整理已有证据；真正的新闻采集、来源核验和影响链推理由 Codex agent 完成。

## 7. Obsidian 工作区

用 Obsidian 打开项目下的 `obsidian-vault/` 文件夹，建议安装 Dataview 插件。

| 文件 | 内容 |
|---|---|
| `_index.md` | 全部股票概览 |
| `_行业知识库.md` | 行业列表、产业链、TAM、财务基准 |
| `_跟踪面板.md` | 跟踪指标与事件覆盖情况 |

当前 vault 结构：

```text
obsidian-vault/
├── _index.md
├── _行业知识库.md
├── _跟踪面板.md
├── 行业知识库/
├── 个股逻辑/
│   └── <代码-名称>/
│       ├── _index.md
│       ├── 基本面.md
│       ├── 技术面.md
│       ├── 筹码面.md
│       ├── 情绪面.md
│       ├── 投资决策.md
│       └── 原始报告/
├── 跟踪面板/
├── 组合管理/
└── 日报/
```

## 8. 自测与排障

常规自测：

```bash
python tools/self-test.py
```

需要检查外部在线数据源时：

```bash
python tools/self-test.py --online
```

自测会检查依赖包、MCP 配置、服务器顶层导入、Skill/命令入口、Python 工具、日报本地链路和 vault 模板结构。若依赖缺失，先重新执行 `pip install -r requirements.txt`。
