# A股AI投研系统 使用手册

> 版本：v0.4.0
> 运行环境：Codex + MCP + Obsidian
> 适用人群：买方投资分析师、个人研究者

## 一、系统概述

本系统是一套运行在 Codex 中的多智能体投研协作平台。它模拟买方投研团队的完整工作流：四个领域分析师并行研究，数据 QA 交叉验证，首席分析师合成结论，最终沉淀为 Markdown 研报与 Obsidian 知识库。

核心原则：研究驱动决策，纪律约束情绪，四维交叉验证，数据支撑每一个观点。

## 二、团队角色

| 角色 | 核心问题 | 关键产出 |
|------|---------|---------|
| 基本面分析师 | 这门生意值不值得投？ | 行业产业链、增长逻辑、财务验证、估值判断 |
| 筹码面分析师 | 大资金在买还是卖？ | 股东结构、资金流、北向/融资、龙虎榜、筹码分布 |
| 技术面分析师 | 什么价买、什么价卖？ | 多周期趋势、指标信号、支撑阻力、交易锚点 |
| 情绪面分析师 | 现在该贪婪还是恐惧？ | 市场情绪、个股舆情、散户逆向、情绪温度计 |
| 首席分析师 | 如何综合决策？ | 编排、矛盾检测、市况加权、投资建议 |
| 数据 QA | 数据是否可靠？ | 缺口清单、交叉验证、通行证 YAML |
| AI 美化师 | 报告是否可读？ | HTML 报告 |
| 组合管理员 | 研究记录如何沉淀？ | 白名单、持仓、黑名单研究记录 |

## 三、工作流

```
用户发起分析
      │
      ▼
首席分析师判断周期
      │
      ├── /analyze-initial
      ├── /analyze-weekly
      ├── /analyze-monthly
      ├── /analyze-quarterly
      ├── /analyze-annual
      └── /analyze-daily
      │
      ▼
四个分析师并行调用 MCP 与公开信息
      │
      ▼
数据 QA 交叉验证
      │
      ▼
首席分析师合成研报
      │
      ▼
tools/archive-to-vault.py 或 tools/update-workspace.py 入库 Obsidian
```

1. 安装依赖：

```bash
pip install -r requirements.txt
```

2. 可选：配置历史兼容 API Key：

```text
```

旧解析模块优先级：`ANTHROPIC_API_KEY` > `DEEPSEEK_API_KEY`。

3. 检查 MCP 配置：

项目根目录的 `.mcp.json` 是 Codex 使用的 MCP 配置，包含：

- `finance-data`
- `tech-analysis`
- `official-announcement`
- `macro-policy`
- `market-news`
- `industry-data`
- `akshare`（默认 disabled）

如更换 Python 或移动项目路径，需要同步更新 `.mcp.json` 中的 `command` 与 `args`。

4. 运行自测：

```bash
python tools/self-test.py
```

## 六、报告与入库

研报默认写入：

```text
reports/stocks/{代码}-{名称}/{周期}/
```

投研完成后可运行：

```bash
python tools/update-workspace.py
```

外部研报或指定目录可直接入库：

```bash
python tools/archive-to-vault.py <报告路径> <代码>-<名称> <行业名> --emit-plan reports/archive-plan-<代码>.json
# Codex 填写 archive-plan 后：
python tools/archive-to-vault.py <报告路径> <代码>-<名称> <行业名> --plan reports/archive-plan-<代码>.json
```

日报使用独立链路：

```bash
python tools/daily_collect.py --date YYYY-MM-DD
python tools/daily_score.py reports/daily/YYYY-MM-DD/evidence.json
python tools/daily_report.py reports/daily/YYYY-MM-DD/scoring.json
python tools/daily_to_vault.py reports/daily/YYYY-MM-DD/日报.md
```

日报默认分析指令发出当天及上一天，即上一日 00:00:00 至当天 23:59:59，并入库 `obsidian-vault/日报/YYYY-MM-DD.md`。日报不直接改写个股四面逻辑，是新闻整理，不默认调用行情、资金、估值或技术指标 MCP。

## 七、Obsidian 工作区

Obsidian 工作区是投研数据的可视化中心。用 Obsidian 打开项目目录下的 `obsidian-vault/`，安装 Dataview 插件后即可查看面板。

| 文件 | 内容 |
|------|------|
| `_index.md` | 全部股票概览 |
| `_行业知识库.md` | 行业列表、产业链、TAM、财务基准 |
| `_投资逻辑.md` | 多空逻辑排行 |
| `_跟踪面板.md` | 跟踪指标与事件覆盖情况 |

每只股票对应一个个股目录，按基本面、技术面、筹码面、情绪面拆分；跟踪任务独立放在 `跟踪面板/<代码-名称>.md`。初次覆盖完整建立基线，后续跟踪默认只在 `投资决策.md` 记录本次结论并指向原文；只有重大逻辑变化才更新相关分面或行业知识库。`原始报告/` 由 Python 自动复制 reports 中的 Markdown 原文，不经过 AI 分类。

```text
obsidian-vault/
├── _index.md
├── _行业知识库.md
├── _投资逻辑.md
├── _跟踪面板.md
├── 行业知识库/
├── 个股逻辑/
│   └── <代码-名称>/
│       ├── _index.md
│       ├── 基本面.md
│       ├── 技术面.md
│       ├── 筹码面.md
│       ├── 情绪面.md
│       ├── 投资决策.md
│       └── 原始报告/
├── 跟踪面板/
│   └── <代码-名称>.md
├── 组合管理/
├── stocks/
└── industries/
```
