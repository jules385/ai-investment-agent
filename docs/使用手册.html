一、系统概述

本系统是一套运行在 Claude Code 上的多智能体投研协作平台。它模拟买方投研团队的完整工作流——从行业判断到买卖决策，由 7 个 AI 角色分工协作完成。

研究驱动决策，纪律约束情绪。四维交叉验证，数据支撑每一个观点。

适合谁用

  • 个人投资者需要系统化的投研流程
  • 买方分析师需要快速覆盖多标的
  • 想用 AI 辅助而非替代投资决策的人

二、团队角色

系统共有 7 个 AI 角色，分为分析层、协调层和展示层：

2.1 分析层（4 位专业分析师）

🏭

基本面分析师

"这生意值不值得投？"
行业判断→增长逻辑(供给/需求/竞争)→财务验证→白名单评级
🌡️

情绪面分析师

"该贪婪还是恐惧？"
情绪温度计(0-100)+左侧/右侧策略建议
🎰

筹码面分析师

"大资金在买还是卖？"
资金合力方向+对手盘压力+建仓/出货阶段
📈

技术面分析师

"什么价买、什么价卖？"
买卖信号+价格锚点(买入/止损/止盈)

2.2 协调层（2 位）

角色职责何时工作
总分析师编排四位分析师并行工作，矛盾检测，市况自适应加权，输出最终投资建议所有分析报告完成后
数据分析师逐条检查报告数据充分性，自主补充缺失数据，生成缺口清单，用户补数据后路由回原分析师修订每份报告完成后、总分析师合成前

2.3 展示层（1 位，按需调用）

角色职责调用方式
AI美化师将 Markdown 报告转化为高可读性 HTML，完整保留原文内容用户说"美化一下"时才工作

三、工作流全景

1
用户发起分析请求
"对002414做初次覆盖" / "对300502做周度跟踪"
2
总分析师判断周期类型
初次覆盖 / 周度 / 月度 / 季度 / 半年 / 年度 → 确定参与分析师和深度
3
四位分析师并行工作
Subagent 并行调用，各自从 MCP/WebSearch 获取数据，输出标准化信号
4
数据分析师 QA
逐条检查数据充分性 → 自主补充缺失 → 缺口清单提交用户
5
总分析师合成
矛盾检测 → 市况自适应加权 → 投资建议(价格+仓位+止损)
6
输出报告
Markdown 存档 →（用户说"美化"后）→ HTML 可视化报告

四、两大工作轨道

Track A：初次覆盖

目标：判断股票是否进入投资组合白名单

1
行业判断
产业链图谱 + TAM/SAM/SOM + 驱动力与瓶颈 → 不通过则终止覆盖
2
增长逻辑分析
供给端/需求端/竞争格局 → 三端信号打分(-3~+3)，遵循《增长逻辑分析方法手册》
3
财务验证
3-5年趋势 + 财务-逻辑一致性检验
4
估值判断
PE分位数 + 可比估值 + DCF区间
5
白名单决策矩阵
六维加权 → 🟢核心(≥4.0) / 🟡卫星(≥3.0) / ⚪观察(≥2.0) / 🔴排除(<2.0)

Track B：持续跟踪（五级周期体系）

周期频率核心问题参与分析师
周度每周六"这周发生了什么需要行动的事？"全四维（基本面为边际监控）
月度每月初"中期趋势强化还是弱化？"全四维
⭐ 季度财报季后"财报验证还是证伪了投资逻辑？"全四维（重点财务验证）
半年年中年末"行业格局有没有结构性变化？"全四维（重点逻辑重验）
年度年末"今年的α来自哪里？明年机会在哪？"全四维（全面刷新）

六、投资组合结构

层级名称数量仓位上限定义
🟢 核心层核心持仓3-5 只单只 ≤20%高确信度、长持有期（评分 ≥4.0）
🟡 卫星层卫星持仓5-10 只单只 ≤10%中短期机会、主题投资（评分 3.0-3.9）
⚪ 观察层观察名单不限0%研究覆盖中、等待买点（评分 2.0-2.9）
🔴 黑名单排除—0%存在致命缺陷（评分 <2.0）

九、核心优势

🔄 多维度交叉验证

四位分析师从完全不同的数据源和分析框架独立得出结论。基本面看"值不值"，情绪面看"怕不怕"，筹码面看"谁在买"，技术面看"何时买"。

📐 数据驱动决策

供给端/需求端/竞争格局共 40+ 量化指标及分析阈值。每个观点必须有数据支撑，每个趋势必须有 3-5 年历史数据验证。

📋 标准化流程，可回溯复盘

每次分析遵循固定工作流，每个标的独立存档。决策日志记录每次操作理由。

🆓 零成本数据基础设施

全部基于 AKShare 免费开源数据，无需 Bloomberg/Wind 等昂贵终端。

🧬 模块化可扩展

7 个分析师 Skill + 63 个金融 Skill + 2 个 MCP Server，均为独立模块。

📚 知识沉淀与复用

每个标的以标准化格式存档，形成可检索的研究知识库。
