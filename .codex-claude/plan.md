# v0.2.0 开发计划：HTML 前端 + 研究员工作区

> Claude 设计架构 → Codex 逐检查点执行 → Claude 审查

## 变更概要

为 A股AI投研系统 增加 HTML 前端界面，实现两大功能板块：
1. **AI 投研对话** — 基于项目 Skills/MCP 的 Claude 对话界面
2. **研究员工作区** — 将研究报告产出内化为结构化的个人研究知识体系

---

## 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                    HTML 前端 (单页应用)                    │
├─────────────────────┬───────────────────────────────────┤
│   ① 投研对话         │   ② 研究员工作区                    │
│   (聊天界面)         │                                   │
│                     │  ┌─ 行业知识库 (按行业分类)         │
│  - Claude 对话窗口   │  │  从研报中提取的客观事实和数据     │
│  - MCP 工具调用展示   │  │                                │
│  - 报告附件预览      │  ├─ 投资逻辑看板 (按标的分类)       │
│  - 命令快捷入口      │  │  多空逻辑 + 关键假设 + 验证状态   │
│                     │  │                                │
│                     │  └─ 边际变化追踪 (按时间排序)        │
│                     │     跟踪指标 + 事件日志 + 预警       │
├─────────────────────┴───────────────────────────────────┤
│                   数据层 (JSON 文件)                       │
│  reports/stocks/ → 解析 → knowledge/{行业}/              │
│                           theses/{标的}/                  │
│                           tracking/{标的}/                │
└─────────────────────────────────────────────────────────┘
```

## 前端技术选型

- 纯 HTML + CSS + Vanilla JS（零依赖，直接双击打开）
- 复用项目现有暗色主题 CSS 变量
- 数据存储在 `reports/` 下的 JSON 文件中（用户本地，不上传）
- Claude API 调用通过后端代理（Python Flask/FastAPI 轻量服务）

**目标**：解析 `reports/stocks/` 下已有的 Markdown 研报，提取三类数据。

**任务清单**：
- [ ] `web/api/parser.py`：`KnowledgeExtractor` 类 — 从研报中提取行业事实

  - 识别 §3.1 5年财务指标表 → 结构化 JSON
- [ ] `web/api/parser.py`：`ThesisExtractor` 类 — 从研报中提取多空逻辑

  - 识别 §2 三端分析中的关键假设

- [ ] `web/api/parser.py`：`TrackerExtractor` 类 — 从研报中提取跟踪框架
  - 识别 §5.5 跟踪指标清单表

  - 识别操作建议
- [ ] `web/api/server.py`：新增 `/api/parse-all` 端点，返回三类数据的 JSON

**验证标准**：
1. 对已有的 002463-沪电股份 初次覆盖报告运行解析，输出 JSON 结构完整
2. 产业链拆分正确识别了 15 个子环节
