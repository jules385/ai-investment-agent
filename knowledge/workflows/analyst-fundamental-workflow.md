# 基本面分析师（Fundamental Analyst）工作流

> **角色定位**：买方行业研究员 —— 判断"这个生意好不好、值不值这个价"。
> **核心能力**：行业增长逻辑 → 增长逻辑分析(供给端/需求端/竞争格局) → 财务交叉验证 → 估值判断
> **关键产出**：白名单评级（初次覆盖）/ 投资逻辑验证结论（跟踪）/ 标准化信号（所有周期）

---

## 一、工作模式总览

基本面分析师有**两种工作模式**，由总分析师根据跟踪周期指定：

```
┌─────────────────────────────────────────────────────────┐
│                    基本面分析师双模式                     │
├─────────────────────────┬───────────────────────────────┤
│  模式A：深度分析模式     │  模式B：边际变化监控模式        │
│  适用：初次覆盖/季度/    │  适用：周度/月度               │
│        半年/年度         │                               │
│  耗时：45-90分钟/标的    │  耗时：5-10分钟/标的            │
│  目标：全面评估 + 估值   │  目标：发现是否影响投资逻辑     │
└─────────────────────────┴───────────────────────────────┘
```

### 1.1 模式触发条件

| 跟踪周期 | 模式 | 深度等级 | 说明 |
|---------|------|---------|------|
| 初次覆盖 | 🅰️ 深度 | ★★★★★ | 全流程五步分析，产出白名单评级 |
| 周度 | 🅱️ 边际监控 | ★★☆☆☆ | 只检查行业/个股重大新闻公告 |
| 月度 | 🅱️ 边际监控 | ★★☆☆☆ | 汇总当月重要基本面事件 |
| 季度 | 🅰️ 深度（财务核心） | ★★★★☆ | 聚焦财报验证 + 估值更新 |
| 半年 | 🅰️ 深度（逻辑验证） | ★★★★★ | 行业格局 + 竞争地位重新审视 |
| 年度 | 🅰️ 深度（全面刷新） | ★★★★★ | 等同于微型初次覆盖 |

## 核心规则（所有模式通用）

### 规则 1：定性分析必须由数据佐证

> 任何定性判断，必须有数据或公开信息来源作为支撑。不得凭空做出"护城河强""行业前景好"等判断。

- 每个定性结论必须附带 **至少一条具体证据**（数据、公告原文、研报引用、行业白皮书等）
- 如果某项关键分析无法获取充足数据，**不得强行给出结论**。必须在报告中明确标注：
  ```
  ⚠️ 数据缺口：[具体缺什么数据]
  建议：[用户如何获取，如"需Wind终端查询XX"或"需购买XX行业报告"]
  ```
- 数据缺口不影响其他维度的分析，但总分析师在做决策时应对该维度降权或标注不确定性。

### 规则 2：使用趋势数据而非时点数据

> 凡是涉及数据论证的，不使用单一时间点的数据。尽可能获取过去 **3-5 年**的历史数据，用趋势来验证或证伪逻辑。

- 财务指标：至少拉取近 3 年数据（年报）+ 最新季度
- 行业数据：至少拉取近 3 年历史 + 未来 3 年预测
- 估值指标：给出当前值 + 近 5 年历史分位数
- 如果某指标仅有当年数据（如新上市的细分赛道），标注"数据年限不足，结论置信度降低"
- 趋势分析的核心问题是：**"这个趋势是可持续的还是周期性的？"**

### 规则 3：定性结论必须展示完整因果推导链

> **数据 ≠ 逻辑。** 不允许从一个初始事实直接跳到最终结论。每个定性结论必须展示从"为什么"到"所以呢"的完整推理过程。

#### 3.1 因果链条层级要求

| 结论层级 | 最低推导步数 | 示例 |
|---------|:--:|------|
| 一级结论（核心投资逻辑/行业驱动力） | ≥4步 | "AI算力需求爆发""毛利率将持续改善" |
| 二级结论（维度判断） | ≥3步 | "公司LPO技术领先""客户粘性强" |
| 三级结论（辅助判断） | ≥2步 | "泰国工厂达产将降本""存货属战略备货" |

#### 3.2 因果链写作模板

```
### [结论名称]

#### 因果推导

事实起点：[公认的事实或第一性原理]

  ↓ 因为[逻辑关系1]

第1步推论：[中间结论A]
  数据验证：[验证A已发生的证据]

  ↓ 因为[逻辑关系2]

第2步推论：[中间结论B]
  数据验证：[验证B已发生的证据]

  ↓ 因为[逻辑关系N]

最终结论：[断言]
  数据验证：[支持最终结论的关键数据]

#### 逻辑脆弱点
最薄弱的环节：[第X步]
如果被打破：[具体情景] → 结论需修正为：[修正后结论]
监控指标：[可观测的先行指标，用于判断逻辑是否失效]
```

#### 3.3 典型错误与修正

```
❌ 错误写法：
"受益于AI算力需求爆发。数据支撑：四大CSP资本开支超6000亿$。"
问题：6000亿$是结果不是原因，因果链完全缺失。

✅ 正确写法：
事实起点：Scaling Law（模型参数量每18个月×10）
  ↓ 因为：更大模型需要更多GPU互联
第1步：GPU集群从千卡→十万卡（验证：xAI集群10万GPU）
  ↓ 因为：GPU算力×5，互联带宽须同步
第2步：互联从400G→800G→1.6T（验证：Rubin标配1.6T）
  ↓ 因为：CSP被迫持续追加AI资本开支
第3步：CSP资本开支超级周期（验证：2025年4600亿$→2026年6000亿$）
  ↓ 因为：GPU与光模块配套比1:3
最终结论：光模块CAGR 30%+
逻辑脆弱点：第2→第3步，若CSP削减AI开支，CAGR降至10-15%
```

#### 3.4 可证伪性要求

每个一级结论必须附带一个**可证伪条件**：

```
如果[可观测的指标]发生[方向性变化]，则该结论需要重新评估。

示例：
"如果英伟达GPU交付周期从当前6个月缩短至1个月（供过于求），
 则AI算力需求可能已阶段性饱和，需重新评估光模块增长假设。"
```

#### 3.5 可以不使用强制因果链的情形

- 直接引用公开数据（如"2025年营收248亿，来源：年报"）
- 纯定量计算（如"PE = 股价/每股收益"）
- 行业常识性事实（如"光模块用于数据中心内部互联"）

### 规则 4：标题必须是该章节的核心结论

> **禁止使用结构标签（"行业分析""增长逻辑分析""财务验证"）作为标题。** 标题必须包含具体数据和结论判断，让读者仅扫目录就能理解完整投资故事。

#### 4.1 标题公式

```
一级标题（H2）：[数据/趋势] + [结论判断]，20-30字，至少含1个关键数字
二级标题（H3）：[论据/数据] + [含义]，15-20字，三端分析标注信号(+1/0/-1)
```

#### 4.2 标题中的禁用词和必须词

| 禁用词（虚标签） | 必须包含 |
|-----------------|---------|
| 分析、判断、概述、研究、关于、探讨 | 具体数字、因果连词（驱动/导致/验证/反映）、方向判断 |

#### 4.3 标题写作程序

```
写完正文后：
1. 用一句话概括本章核心结论
2. 压缩到20-30字，保留最重要的数字
3. 检查是否用了禁用词（"分析""判断"等）→ 有则回到第1步
4. 连读所有一级标题 → 确认能形成完整的投资故事线 → 不能则修改
```

#### 4.4 正确与错误示例

```
❌ 一、行业判断
✅ 一、AI算力驱动光模块五年超级周期，全球市场CAGR 30%+

❌ 1.1 产业链分析
✅ 1.1 中游光模块制造价值最集中，新易盛卡位核心环节

❌ 二、增长逻辑分析
✅ 二、产能扩张×需求渗透×格局优化：三端共振，增长逻辑最强

❌ 2.1 供给端分析
✅ 2.1 产能翻3倍+毛利率47.8%：供给端扩张与效率双升（信号+1）

❌ 三、财务验证
✅ 三、营收+187%、ROE 73%、现金流77亿：财务全面验证增长逻辑
```

## 二、模式 A：深度分析流程（五步递进法）

```
Step 1        Step 2              Step 3        Step 4        Step 5
行业判断  →  增长逻辑分析    →  财务验证  →  估值判断  →  白名单/逻辑结论
 35min       (供给端/需求端/    45min         30min         15min
             竞争格局) 50min
   │            │                  │            │            │
   └── 不通过？终止 ──┘     └── 矛盾？回溯 ──┘    └── 输出标准化信号
```

### Step 1：行业增长逻辑判断（35分钟）

#### 1.1 行业定性分析

##### 1.1.1 产业链分析（必须完成）

> ⚠️ **拆分深度要求**：每个游至少拆出3个子环节。子环节 ≠ 公司名单——必须包含名称/功能/毛利率区间/壁垒类型/代表公司。成熟业务与新兴业务必须区分。

**Step 1：拆分子环节**

```
产业链拆分（每个游 ≥3 子环节）：

上游
├── [子环节1]：[功能描述]
│   ├── 代表：[公司A]、[公司B]
│   └── 特征：毛利率[X%]，壁垒[类型]，[痛点/机会]
├── [子环节2]：...
└── [子环节3]：...

中游
├── [子环节1]：...           ← 如为成熟业务标注
├── [子环节2]：...           ← **标的所在子环节（加粗）**
└── [子环节3]：...           ← 如为下一代方向标注

下游
├── [子环节1]：[客户类型A]
├── [子环节2]：[客户类型B]
└── [子环节3]：[客户类型C]
```

**Step 2：产业链价值排序（必须在标的定位之前完成）**

使用统一标准对全产业链所有子环节进行独立评分排序：

| 指标 | 权重 | 说明 |
|------|:--:|------|
| 市场增速（子环节CAGR） | 30% | 高速增长环节天然更有投资价值 |
| 利润率（毛利率中枢） | 25% | 高毛利环节通常壁垒更深 |
| 集中度（CR3） | 20% | 集中度高的环节龙头溢价能力强 |
| 壁垒强度（定性） | 15% | 技术/资质/客户认证等进入壁垒 |
| 国产替代空间 | 10% | 国产化率越低，替代空间越大 |

```
产业链价值排序（先于标的分析）：

| 排名 | 子环节 | 所属游 | CAGR | 毛利率 | CR3 | 壁垒 | 得分 | 标的所在？ |
|:--:|--------|:--:|------|--------|-----|------|:--:|:--:|
| 1 | [环节A] | 上游 | X% | X% | X% | 技术 | X.X | 否 |
| 2 | [环节B] | 中游 | X% | X% | X% | 认证 | X.X | **是** |
| 3 | [环节C] | 下游 | X% | X% | X% | 规模 | X.X | 否 |
```

**判断规则**：

```
如果标的所在环节排名第1 → 正常表述
如果标的所在环节不排第1 → 必须说明：
  "虽然[标的]处于排名第X的[环节B]，但其在子环节内的竞争优势
   [具体优势]使其仍具备投资价值。需注意该环节的结构性限制：[具体限制]"

🔴 禁止行为：
  - 不得仅因标的身处某环节而将该环节评为最具价值（循环论证）
  - 不得为让标的所在环节排第一而调整评分标准或数据
  - 不得用模糊语言回避排名（如"各环节各有千秋"而不给出明确排序）
```

> **数据要求**：产业链信息优先从券商行业深度研报中获取（已有研究交叉验证后可采纳），辅以 WebSearch 确认。

##### 1.1.2 驱动力与瓶颈（各限3条以内）

```
核心驱动力：
  1. [驱动力1] — 确定性：高/中/低 — 数据支撑：[具体数据]
  2. [驱动力2] — 确定性：高/中/低 — 数据支撑：[具体数据]
  3. [驱动力3] — 确定性：高/中/低 — 数据支撑：[具体数据]

核心瓶颈：
  1. [瓶颈1] — 严重程度：高/中/低 — 数据支撑：[具体数据]
  2. [瓶颈2] — 严重程度：高/中/低 — 数据支撑：[具体数据]
  3. [瓶颈3] — 严重程度：高/中/低 — 数据支撑：[具体数据]
```

#### 1.2 市场空间定量分析（必须完成）

> ⚠️ 本部分要求尽可能获取 **过去3-5年实际数据** 以及 **未来3/5/10年预测数据**，用趋势验证增长逻辑。

```
市场空间测算：

┌──────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│ 指标  │ Y-2实际   │ Y-1实际   │ 最新实际   │ Y+3预测   │ Y+5预测   │
├──────┼──────────┼──────────┼──────────┼──────────┼──────────┤
│ TAM  │ X亿元     │ X亿元     │ X亿元     │ X亿元     │ X亿元     │
│ SAM  │ X亿元     │ X亿元     │ X亿元     │ X亿元     │ X亿元     │
│ SOM  │ X亿元     │ X亿元     │ X亿元     │ X亿元     │ X亿元     │
└──────┴──────────┴──────────┴──────────┴──────────┴──────────┘

TAM（总可寻址市场）：[定义 + 规模]
SAM（可服务市场）：[定义 + 规模，剔除公司无法触及的细分]
SOM（可获得市场）：[定义 + 规模，基于公司当前市占率和能力边界]

测算公式与依据：
  - TAM = [公式]，数据来源：[机构研报/行业白皮书/统计局]
  - SAM = [公式]，数据来源：[同上]
  - SOM = [公式]，数据来源：[同上]

机构预测交叉验证：
  - 来源A（[券商名]研报）：TAM 2026E = [X亿]，CAGR = [X%]
  - 来源B（[券商名]研报）：TAM 2026E = [X亿]，CAGR = [X%]
  - 交叉验证结果：[一致/存在分歧，说明差异原因]
  - 采纳方案：[如分歧较小取均值；如分歧大，说明本报告采纳哪一方及理由]
```

> **数据缺口处理**：如果 SAM/SOM 无法精确测算（非常见情形），标注"⚠️ SAM/SOM 数据不可得，仅以 TAM 和公司市占率趋势替代"，不得编造数据。

#### 1.3 行业定性综合输出

```
行业分析结论：

1. 行业定义：[一句话]
2. 行业生命周期：[导入期/成长期/成熟期/衰退期]
3. 政策环境：[友好/中性/不利] — [政策要点]
4. 行业集中度：[分散/集中/寡头] — CR5=[X%]，趋势：[集中化/分散化/稳定]
5. 行业核心风险：[最大风险]

行业判断：✅ 值得深入 / ❌ 终止覆盖
如果 ❌，理由：[一句话]
```

### Step 2：增长逻辑分析 —— 供给端 / 需求端 / 竞争格局（50分钟）

> ⚠️ **本步骤必须使用**《增长逻辑分析方法手册》（同目录下 `增长逻辑分析方法手册.md`）中定义的分析框架。
> 分析前必须先读取该手册。本 Step 是对手册三端框架的具体执行流程。

#### 手册核心框架速查

```
供给端（公司能否持续提供有竞争力的产品或服务？）
  ├── 产能与扩张能力：产能利用率 / 在建工程占比 / 资本开支强度
  ├── 成本控制与供应链：毛利率趋势 / 单位成本 / 原材料敏感性 / 存货周转
  ├── 技术壁垒与研发效率：研发费用率 / 资本化率 / 专利引用 / 技术人员流失率
  └── 交付能力：订单交付周期 / 客诉率

需求端（产品/服务能否持续被更多/更贵地购买？）
  ├── 市场空间与渗透率：TAM/SAM/SOM / 渗透率 / 客单价趋势
  ├── 客户结构及粘性：集中度 / 复购率 / LTV/CAC / DAU/MAU
  ├── 定价能力：提价后销量变化 / 毛利率 vs 行业 / 终端价格
  └── 渠道与区域：单店收入(SSSG) / 渠道库存 / 海外收入占比

竞争格局（护城河能否维持或扩大？）
  ├── 市场份额：市占率趋势 / CR3/CR5/HHI / 市占率来源分析
  ├── 护城河量化：ROIC-WACC / 超额利润率持久性 / 无形资产
  ├── 上下游议价：应付vs应收天数 / 合同负债 / 净运营资本占比
  └── 进入者威胁：行业资本开支门槛 / 跨界技术渗透 / 政策壁垒
```

#### 2.1 数据获取（按三端组织）

| 分析维度 | 核心数据需求 | 来源 |
|---------|------------|------|
| 供给端-产能 | 产能利用率、在建工程、资本开支 | 年报附注 + 券商研报 |
| 供给端-成本 | 毛利率(分季度)、单位成本 | AKShare `get_financials` + 年报 |
| 供给端-研发 | 研发费用率、资本化率、技术人员 | 年报 + WebSearch 专利数据 |
| 需求端-空间 | TAM/SAM/SOM、渗透率 | 行业研报 + 第三方报告 |
| 需求端-客户 | 前五大集中度、客单价趋势 | 年报 + AKShare |
| 需求端-粘性 | 复购率/NDR、DAU/MAU | 招股书/研报/QuestMobile |

```

#### 3.4 财务-逻辑一致性检验

```
关键验证问题：
1. 毛利率趋势是否与供给端(成本控制)和竞争格局判断一致？
   ├── 成本优势/品牌溢价 + 毛利率稳中有升 → ✅ 自洽
   └── 声称有壁垒 + 毛利率持续下滑 → ⚠️ 矛盾！查找原因

2. 营收增速是否与行业增速匹配？
   ├── 营收增速 > 行业增速 → ✅ 份额在扩大
   └── 营收增速 < 行业增速 → ⚠️ 份额被侵蚀

3. 经营现金流是否持续健康？
   ├── 经营现金流/净利润 > 0.8 → ✅ 利润有现金支撑
   └── 经营现金流/净利润 < 0.5 → ⚠️ 利润质量差

4. 应收账款增速是否快于营收增速？
   ├── 应收账款/营收 稳定 → ✅ 回款正常
   └── 应收账款增速 > 营收增速 → ⚠️ 可能放松信用政策冲收入

5. 研发投入是费用化还是资本化？
   ├── 研发费用化率高 → ✅ 保守，利润质量好
   └── 研发资本化率高 → ⚠️ 激进，可能美化利润

一致性结论：[自洽 / 存在矛盾，说明]
```

不需要重做：Step 1 行业全量分析、Step 2 增长逻辑全量评估（除非有明显变化信号）
```

### 半年

```
模式：深度（逻辑验证）★★★★★
Step 1-5 全部执行，特别重点：
  - 行业格局是否结构性变化？
  - 竞争地位是上升还是下降？
  - 盈利预测是否需要大幅修正？
  - 估值中枢是否变化？
  - 白名单评级是否需要调整？
```

### 年度

```
模式：深度（全面刷新）★★★★★
等同于微型初次覆盖
额外增加：
  - 全年盈利预测准确度复盘
  - 投资逻辑偏差分析
  - 下一年度核心假设制定
```
