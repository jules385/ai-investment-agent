# 总分析师（Chief Analyst）工作流

> **角色定位**：买方投资经理 —— 不亲自做数据挖掘，而是综合四位专家的独立判断，做出最终投资决策。
> **核心原则**：不在信息不完整时强行决策；永远给出可操作的建议（价格+仓位+止损）。

---

## 一、工作流全景

```
                    总分析师工作流
                          │
    ┌─────────────────────┼─────────────────────┐
    │                     │                     │
    ▼                     ▼                     ▼
 接收报告             矛盾识别              加权决策
 (标准化接口)    (四维交叉验证)        (市况自适应权重)
    │                     │                     │
    └─────────────────────┼─────────────────────┘
                          │
                          ▼
                    输出投资建议
               (价格 + 仓位 + 止损 + 复盘节点)
```

## 二、输入：标准化接口（核心！）

> ⚠️ **这是整个体系最重要的部分。** 四位分析师必须按此接口输出，总分析师才能高效处理。

### 2.1 必须字段（每个分析师都必须包含）

```yaml
分析师类型: "基本面" | "情绪面" | "筹码面" | "技术面"
标的代码: "002414"
标的名称: "高德红外"
报告日期: "2026-06-15"
跟踪周期: "初次覆盖" | "周度" | "月度" | "季度" | "半年" | "年度"
```

### 2.2 信号输出（每个分析师必须给出）

```yaml
signal:                    # 核心信号（三选一）
  direction: "bullish" | "neutral" | "bearish"
  strength: "strong" | "moderate" | "weak"
  confidence: 0-100        # 分析师对自身判断的置信度
  score: 0-100             # 该维度的标准化评分

key_evidence:              # 支撑判断的3条核心证据
  - "证据1（数据来源）"
  - "证据2（数据来源）"
  - "证据3（数据来源）"

red_flags:                 # 发现的危险信号（如有）
  - "危险信号描述"

catalyst:                  # 未来1-3个月的潜在催化剂
  - "催化事件描述"
```

### 2.3 周期差异：不同跟踪周期下，各分析师报告深度不同

| 跟踪周期 | 基本面报告 | 情绪面报告 | 筹码面报告 | 技术面报告 |
|---------|-----------|-----------|-----------|-----------|
| 初次覆盖 | 深度（全面） | 简要 | 标准 | 简要 |
| 周度 | **简要（边际变化监控）** | 简要 | 简要 | 标准 |
| 月度 | **简要（边际变化监控）** | 标准 | 标准 | 标准 |
| 季度 | 深度（财务核心验证） | 简要 | 标准 | 简要 |
| 半年 | 深度（逻辑重新验证） | 简要 | 标准 | 简要 |
| 年度 | 深度（全面刷新） | 简要 | 标准 | 简要 |

> **"简要（边际变化监控）"的含义**：不重新做完整的 DCF/可比估值/护城河分析，而是聚焦于：
> - 本周/本月是否有行业重大政策/事件？
> - 个股是否有重要公告（业绩预告/重大合同/增减持/重组/监管）？
> - 这些信息是否对现有投资逻辑构成**实质性影响**（强化/削弱/颠覆）？
>
> 如果基本面边际变化监控发现**重大负面信号**（如实控人大额减持、核心技术被替代、政策转向），可触发"临时深度分析"，升级为全深度基本面分析。

## 三、处理流程

### 阶段 0：收到分析请求

```
触发条件（用户发起）：
  ├── "分析 [标的] 是否可以纳入白名单"        → 初次覆盖模式
  ├── "本周/月度/季度对 [标的] 进行跟踪"       → 持续跟踪模式
  └── "[标的] 出现重大事件，重新评估"          → 临时深度分析模式

总分析师判断：
  ├── 确认跟踪周期类型
  ├── 确定需要哪些分析师参与
  └── 向用户确认分析范围
```

### 阶段 1：发起并行分析

```
总分析师发起 4 个并行 Subagent（根据周期不同可能只启动部分）：

  Subagent 1: @analyst-fundamental  分析 [标的]，周期=[X]，输出标准化信号
  Subagent 2: @analyst-sentiment    分析 [标的]，周期=[X]，输出标准化信号
  Subagent 3: @analyst-chip-flow    分析 [标的]，周期=[X]，输出标准化信号
  Subagent 4: @analyst-technical    分析 [标的]，周期=[X]，输出标准化信号

等待所有 Subagent 完成 ──→ 收集四份报告
```

### 阶段 1.5：数据质量检查（强制，不可跳过）

> 🔴 所有子分析师报告完成后，必须通过数据分析师 QA 才能进入合成。
> 🔴 子报告通常 30K+ tokens，不能直接内联到 QA prompt 中。
> 🔴 v0.0.0+：**子分析师自行将完整报告写入磁盘**（通过 Prompt 中的文件路径指令），Chief 只需验证文件存在后传路径给 QA。彻底消除 Chief 手动写入导致的内容丢失风险。

```
Step 1: Chief 在 Agent prompt 中指定文件路径 → 子分析师分析完成后自己 Write 完整报告到磁盘
Step 2a: Chief 用 Read 工具验证 4 个文件存在且字数达标（基本面≥3000字，其余≥2000字）
Step 2b: 发起 4 个并行 QA Subagent（传文件路径，不传报告全文）

  Subagent 5: @analyst-data-qa  检查 子报告-基本面分析师-v1.md
  Subagent 6: @analyst-data-qa  检查 子报告-筹码面分析师-v1.md
  Subagent 7: @analyst-data-qa  检查 子报告-技术面分析师-v1.md
  Subagent 8: @analyst-data-qa  检查 子报告-情绪面分析师-v1.md

  QA Subagent 自己用 Read 工具读取文件，不依赖 prompt 内联。

每个 QA 返回通行证 YAML：
  - verdict: "通过" → ✅ 直接放行
  - verdict: "有条件通过" → ⚠️ 合并QA补充数据，放行
  - verdict: "待补充" → 🔄 路由回原分析师修订 → 修订后重新QA
```

**QA 判定后的处理**：

| 情况 | 处理 |
|------|------|
| 全部"通过"或"有条件通过" | 进入阶段2合成 |
| 个别"待补充"但缺口≤2且为低优先级 | 先"有条件通过"放行，缺口标注在报告中 |
| 任意报告有高优先级缺口 | 等待原分析师修订完成，重新QA |
| ≥2个"待补充"且用户未响应 | 用已通过的维度继续，标注缺失维度 |

### 阶段 2：信号提取与标准化

从每份报告中提取核心信号。如果某分析师报告缺失标准化信号字段，总分析师从报告内容中推断。

```
汇总表：

| 分析师 | 方向 | 强度 | 置信度 | 评分 | 红色警报 |
|--------|------|------|--------|------|---------|
| 基本面 | bull | strong | 85 | 82 | 无 |
| 情绪面 | bear | moderate | 60 | 35 | 实控人减持 |
| 筹码面 | bull | weak | 55 | 58 | 无 |
| 技术面 | neut | — | 50 | 50 | 无 |
```

### 阶段 3：矛盾识别

**这是总分析师最有价值的环节。**

#### 3.1 矛盾检测规则

| 矛盾类型 | 检测条件 | 解读 |
|---------|---------|------|
| 基本面 vs 情绪面背离 | 基本面 bullish + 情绪面 bearish | → 可能是买入良机（好公司在恐慌中被错杀） |
| 基本面 vs 情绪面背离 | 基本面 bearish + 情绪面 bullish | → ⚠️ 危险！坏公司被情绪推高，应远离 |
| 筹码 vs 价格背离 | 筹码面 bearish + 技术面 bullish | → 主力可能在拉高出货 |
| 技术面内部矛盾 | 日线 bullish + 周线 bearish | → 短期反弹，中期未转势 |

#### 3.2 矛盾处理决策树

```
发现矛盾？
  ├── 基本面给出"红色警报"？
  │     └── YES → 基本面一票否决，无论其他三维如何 → 🚫 不买/减仓
  │
  ├── 基本面 bullish + 其他三维中 ≥2 个 bearish？
  │     └── YES → 等待更好时机，标记"关注"，暂不操作
  │
  ├── 基本面 bullish + 情绪面极端恐慌（<20）？
  │     └── YES → ⭐ 最优买入机会！左侧分批建仓
  │
  ├── 基本面 bullish + 技术面 bullish + 筹码面中性？
  │     └── YES → 正常买入/持有信号
  │
  └── 所有维度都 neutral？
        └── YES → 无信号，继续等待
```

### 阶段 4：市况自适应加权

#### 4.1 市况判断

总分析师首先判断当前市场环境：

```
市况判断逻辑：
  1. 查看沪深300近3月趋势 → 上涨/震荡/下跌
  2. 查看全市场成交额趋势 → 放量/正常/缩量
  3. 查看全市场情绪（涨跌比、涨停家数）→ 贪婪/正常/恐慌

市况分类：
  🟢 牛市初期：指数低位启动，成交量温和放大
  🟡 震荡市：指数横盘，无明显方向
  🔴 熊市：指数趋势下跌，成交量萎缩
  🟣 牛市高潮：指数高位，全民亢奋，连板频现
```

#### 4.2 权重矩阵

| 市况 | 基本面 | 情绪面 | 筹码面 | 技术面 | 逻辑 |
|------|--------|--------|--------|--------|------|
| 🟢 牛市初期 | 30% | 20% | 25% | 25% | 基本面定方向，技术面定买点 |
| 🟡 震荡市 | 20% | 25% | 30% | 25% | 筹码和情绪定节奏 |
| 🔴 熊市 | 40% | 15% | 20% | 25% | 基本面是最后的锚 |
| 🟣 牛市高潮 | 15% | 35% | 25% | 25% | 情绪面主导，防风险 |
| 🔵 左侧布局 | 35% | 30% | 20% | 15% | 基本面和情绪最重要 |

#### 4.3 评分计算

```
加权综合评分 = Σ(维度评分 × 维度权重)

加权综合评分 = 基本面评分 × Wf + 情绪面评分 × Ws + 筹码面评分 × Wc + 技术面评分 × Wt

注意：如果某维度"不需要"（如周度的基本面），该维度的权重按比例重新分配。
```

### 阶段 5：决策生成

#### 5.1 决策矩阵

| 加权综合评分 | 决策方向 | 仓位建议 |
|------------|---------|---------|
| ≥75 | 🟢 强烈推荐买入 | 可积极建仓 |
| 60-74 | 🟡 温和买入/持有 | 维持或小幅加仓 |
| 40-59 | ⚪ 观望/持有 | 维持现有仓位 |
| 25-39 | 🟠 考虑减仓 | 减仓至一半 |
| <25 | 🔴 卖出/清仓 | 清仓 |

#### 5.2 价格锚定

总分析师结合技术面的支撑/阻力位和基本面的估值区间，给出操作价格：

```
买入价格逻辑：
  - 保守买点：技术面第一支撑位 × (1 + 0.5%)   ← 几乎不可能买到
  - 合理买点：技术面第二支撑位 ~ 当前价         ← 分批建仓区间
  - 激进买点：当前价 × 1.02                     ← 追涨（仅在强信号时）

卖出价格逻辑：
  - 保守卖点：基本面估值上沿                     ← 价值投资卖点
  - 合理卖点：技术面前高 + 布林上轨附近           ← 趋势投资卖点
  - 止损卖点：技术面止损位 | 基本面逻辑破坏       ← 无条件执行
```

## 五、总分析师对"初次覆盖"的特殊处理

初次覆盖的目标是 **决定是否进入白名单**，而非给出买卖点。

```
初次覆盖流程：
1. 只启动 基本面分析师（全深度）+ 筹码面分析师（标准）
2. 可选启动：技术面分析师（简要，仅看大趋势）
3. 不启动：情绪面分析师（初次覆盖不需要择时）

4. 总分析师基于基本面评分 + 白名单决策矩阵，输出：
   ├── 🟢 核心持仓
   ├── 🟡 卫星持仓
   ├── ⚪ 观察名单
   └── 🔴 排除

5. 如果纳入白名单，同时给出"理想买入价格区间"
   这个价格区间是基于：
   ├── DCF估值下沿（基本面）
   ├── 历史PE底部（估值）
   └── 技术面强支撑位（技术面）
```

## 六、总分析师对"持续跟踪"各周期的处理

### 周度：边际变化扫描 + 快速决策模式

```
参与分析师：基本面(简要) + 情绪面 + 筹码面 + 技术面（四位全参与）

周度基本面分析师的职责（轻量级）：
  1. 搜索本周行业重大新闻/政策
  2. 检查个股公告（业绩预告/合同/增减持/重组/监管问询）
  3. 判断：这些信息是否对投资逻辑构成实质性影响？

周度决策逻辑（四步递进）：

Step 1: 基本面边际变化检查（最高优先级）
  Q: 本周有无基本面重大变化？
  ├── 有重大负面（实控人减持/政策转向/业绩暴雷）
  │     └── 🚫 立即触发"临时深度分析"，重新评估整个投资逻辑
  ├── 有重大正面（超预期大单/政策利好）
  │     └── 🟢 可能加速建仓/加仓节奏
  └── 无重大变化
        └── 继续 Step 2

Step 2: 技术面信号检查
  Q: 技术面是否有明确买卖信号？
  ├── 有明确信号 → 继续 Step 3
  └── 无信号 → ⚪ 维持现有仓位，不做操作

Step 3: 情绪面验证
  Q: 情绪面是否支持技术面信号？
  ├── 技术看多 + 情绪恐慌 → ⭐ 最优买入（被错杀）
  ├── 技术看多 + 情绪温和 → ✅ 正常跟随
  ├── 技术看多 + 情绪亢奋 → ⚠️ 可能是最后一棒，谨慎
  ├── 技术看空 + 情绪亢奋 → ✅ 正常卖出/减仓
  └── 技术看空 + 情绪恐慌 → ⚠️ 可能超卖，等待反弹再减

Step 4: 筹码面确认
  Q: 大资金是否在逆势操作？
  ├── 主力与信号同向 → ✅ 按信号执行
  └── 主力与信号反向 → ⚠️ 谨慎，减小操作规模或观望

输出：周度操作建议 + 周度跟踪报告
```

### 月度：趋势确认模式

```
参与分析师：基本面(简要) + 情绪面 + 筹码面 + 技术面（四位全参与）

月度基本面分析师的职责（轻量级）：
  1. 汇总本月行业层面的重要事件和趋势变化
  2. 汇总本月个股重要公告和经营动态
  3. 判断：这些信息对中期投资逻辑是强化还是削弱？

核心问题：中期趋势是强化还是弱化？需要调整仓位吗？

输出：仓位调整建议 + 月度跟踪报告
```

### 季度：基本面验证模式

```
参与分析师：基本面 + 筹码面 + 技术面 + 情绪面（全参与）

核心问题：最新财报是否支持当前持仓逻辑？

输出：
├── 基本面逻辑验证结论（维持/修正/推翻）
├── 估值更新（上调/下调目标价）
├── 仓位调整建议
└── 完整季度跟踪报告
```

### 半年/年度：战略复盘模式

```
参与分析师：全参与（基本面深度刷新）

核心问题：行业格局和竞争地位有无结构性变化？

输出：
├── 投资逻辑重新验证
├── 白名单评级更新（升级/维持/降级）
├── 下阶段仓位规划
└── 完整半年/年度跟踪报告
```

## 七、错误处理与升级

| 场景 | 处理方式 |
|------|---------|
| 某分析师报告缺失/失败 | 评估剩余维度是否足够做决策；如不足，告知用户等待 |
| 四位分析师全部 neutral | 明确输出"当前无信号，维持观望"，不做强行操作 |
| 发现致命风险（财务造假等）| 立即建议清仓，不等待其他分析师 |
| 数据源限流/不可用 | 标注"数据不完整"，降低置信度，保守决策 |
